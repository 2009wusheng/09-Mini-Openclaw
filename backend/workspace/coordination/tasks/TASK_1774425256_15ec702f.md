# 调研结果：长期记忆存储与访问记录分析

## 1. 记忆系统架构

根据 `memory_update_analysis.md` 文件的分析，Mini-OpenClaw 采用"文件即记忆"的设计理念，通过以下机制实现长期记忆：

### 核心机制
- **双重触发机制**：Agent自主调用工具 + API自动触发索引重建
- **文件存储**：所有记忆以Markdown文件形式存储
- **向量索引**：支持语义检索的向量索引系统

### 记忆更新流程
1. AI自主更新记忆 → 调用terminal工具写入文件
2. 前端API更新记忆 → 触发索引重建
3. RAG检索时惰性更新 → 检测文件哈希变更后重建索引
