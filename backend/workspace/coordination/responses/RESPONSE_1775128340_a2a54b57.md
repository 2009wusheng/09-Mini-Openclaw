# Mini-OpenClaw 技术核心分析报告

## 一、项目概述

**Mini-OpenClaw** 是一个轻量级、全透明的AI Agent系统后端服务，基于FastAPI + LangChain构建，支持SSE流式输出。项目采用去中心化的多Agent协同架构，具有"文件即记忆"、"技能即插件"的核心设计理念。

## 二、核心技术架构

### 2.1 分层架构设计

```
┌─────────────────────────────────────────────────┐
│                    API层 (FastAPI)              │
│  /api/chat /api/sessions /api/files /api/tokens │
└─────────────────────────────────────────────────┘
                        ↓

└─────────────────────────────────────────────────┘
```

### 2.2 多Agent协同架构

项目采用去中心化的多Agent协同架构，包含三种Agent类型：

1. **Primary Agent** - 用户交互入口，任务拆解与结果汇总
2. **Domain Agent** - 承接单一领域原子子任务
3. **Coordinator Agent** - 协同规则执行，状态同步与冲突解决

**协同特点**：
- 无主从层级，各Agent独立拥有核心工具与技能体系
- 基于标准化文件交互（TASK_{id}.md, RESPONSE_{id}.md）
- 轻量通信，无网络跨进程依赖
- 透明可控，所有协同过程可追溯

## 三、核心模块详解

### 3.1 Agent引擎 (graph/)

**AgentManager** - 核心单例类
- 基于LangChain 1.x的create_agent API构建
- 使用Graph运行时管理Agent生命周期
- 内置流式tool_call兼容性检测与修复

**关键技术特性**：
- **流式兼容性适配**：通过StreamingToolCallAdapter修复阿里云DashScope等平台的流式tool_call聚合问题
- **递归限制**：RECURSION_LIMIT=50防止无限循环
- **空参数检测**：连续3次空参数工具调用判定为流式兼容性问题
- **错误处理**：自动检测Traceback/Exception，标记工具执行状态

**PromptBuilder** - 动态System Prompt构建
- 6部分System Prompt动态拼接：
  1. SKILLS_SNAPSHOT.md - 可用技能清单
  2. workspace/SOUL.md - 人格、语气
  3. workspace/IDENTITY.md - 名称、风格
  4. workspace/USER.md - 用户画像
  5. workspace/AGENTS.md - 操作指南
  6. memory/MEMORY.md - 长期记忆（或RAG检索结果）

**MemoryIndexer** - RAG记忆索引
- 构建MEMORY.md的向量索引（LlamaIndex + BM25混合检索）
- 支持RAG模式开关，动态注入相关记忆片段

### 3.2 工具系统 (tools/)

**5个核心工具**：
1. **terminal_tool** - Shell命令执行（沙箱化，30s超时）
2. **python_repl_tool** - Python代码执行
3. **fetch_url_tool** - 网页内容获取（15s超时）
4. **read_file_tool** - 项目文件读取
5. **search_knowledge_tool** - RAG语义检索 + BM25混合

**安全机制**：
- 危险命令黑名单（rm -rf /, shutdown, sudo等）
- 敏感环境变量过滤
- 路径访问限制在项目目录内
- 超时控制防止资源占用

### 3.3 技能系统 (skills/)

### 3.4 记忆系统 (memory/)

**文件即记忆设计**：
